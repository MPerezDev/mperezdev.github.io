Introducción

En este documento se encuentra la descripción del proceso llevado a cabo en el pipeline para el correcto desarrollo de la práctica, así como el script de Python que ejecutamos en el pipeline.

1. Pipeline

A continuación vamos a describir las distintas partes del pipeline. En primer lugar indicamos la imagen que vamos a usar, así como los distintos stages y las variables que vamos a necesitar.

En las siguientes secciones de la documentación vamos a explicar las distintas partes que tiene el pipeline, desde los comandos previos al desarrollo de la práctica hasta la ejecución del script y la clonación a GitHub.

1.1 Sección “before_script”

En esta sección nos encargamos de actualizar e instalar las librerías necesarias para el script de Python que vamos a ejecutar, además de añadir los datos necesarios de GitHub para la correcta ejecución de la práctica.

1.2 Sección “build”

En esta sección simplemente clonamos el repositorio de Gitlab y ejecutamos el script de Python para generar la gráfica y enviarla a un servicio de alojamiento de datos abierto, en nuestro caso Figshare.

1.3 Sección “documentation”

En esta sección nos encargamos de generar el archivo html a partir de la documentación en el formato markdown gracias a pandoc, además de sincronizar estos cambios con la rama en Github, lo que producirá un nuevo despliegue de Github Pages actualizado gracias a GitHub Actions.

2. Script de Python

En esta parte del documento hablaremos sobre el script que hemos preparado para la ejecución de la práctica. En este caso se nos pide que el script sea en Python y que realice una transformacón de datos como se nos describe en el enunciado.

En primer lugar, importamos las librerías necesarias, así como también declaramos las variables que nos van a hacer falta.

A continuación podemos encontrar algunas funciones que usamos para interactuar con la API de Figshare, como son “raw_issue_request”, “issue_request” y todas las relacionadas con los artículos y las cargas de archivos en la plataforma.

Después tenemos las funciones “cargarDatos” y “obtenerDatosGrafica”, las cuales se encargan de cargar los datos del fichero .scv y de extraer dos columnas concretas, respectivamente.

El último “grupo” de funciones que encontramos son las que se encargan de generar y mostrar la gráfica resultante respectivamente, que son las funciones “crearGrafica” y “guardarMostrarGrafica”.

Finalmente definimos cual va a ser el fichero .csv a utilizar (en este caso es SensorData.csv)y el archivo en el que guardaremos la gráfica resultante, para posteriormente ir recurriendo a las funciones que necesitemos para llevar a cabo la transformación de los datos del fichero en una gráfica que relacione, en nuestro caso, dos variables que hacen referencia a la humedad y la temperatura de la lista de objetos.
